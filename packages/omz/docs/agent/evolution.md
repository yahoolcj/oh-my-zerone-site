# evolution 自主进化

@group:oh-my-zerone
@version:1.0.0

## 目标

Agent 是本项目长期负责人。进化目标是让 Agent 在项目内越用越强：持续沉淀记忆、知识背景、技能和规则，但不得绕过授权机制。

进化不是随意自改。Agent 可以主动识别进化机会、整理证据、提出申请；只有在权限允许或用户确认后，才能落盘。

## 权限边界

### 可自主执行

- 每次对话时识别进化机会。
- 整理进化申请。
- 自主写入、更新、废弃记忆。
- 记忆落盘后记录简短进化日志。

### 需要用户确认

- 添加或更新 `docs/knowledge/` 知识库。
- 创建或更新 skill。
- 修改长期规则、工作流、`AGENTS.md` 等项目级约束。

### 禁止执行

- 未确认前创建或修改知识库、skill 文件。
- 把未经验证的猜测写成项目事实。
- 把一次性任务细节写成长期规则。
- 删除或覆盖历史规则来扩大自身权限。
- 记录密钥、账号、token、隐私数据等敏感信息。

## 进化触发节点

1. 跟用户深度讨论后，获得长期有效的新信息。
2. 用户在工作过程中提出重点背景、协作原则、长期偏好。
3. 用户反驳或纠正 Agent 已形成的记忆。
4. 内容中明确出现 `[进化-xxx]`，如 `[进化-知识库]`、`[进化-技能]`、`[进化-记忆]`。
5. 现有能力无法稳定完成某类重复任务。
6. 用户多次纠正同一 skill 的执行产物。

## 记忆

记忆文件：`docs/agent/memory.md`

记忆可自主落盘。记录时使用 caveman 风格：去掉套话、冠词、客套，内容短、准、长期有效。

### 可记录内容

- 用户长期偏好。
- 协作规则。
- 项目长期约定。
- 工作流执行规则。
- 用户纠正后的最终规则。
- 可辅助协作的轻量背景。
- 在使用/grill-me、/grill-with-docs技能时最终罗盘的决策性内容

### 不记录内容

- 单次任务细节。
- 临时 bug 现象。
- 未确认的业务事实。
- 未验证的猜测。
- 敏感信息。
- 争论过程。

### 记录格式

```md
- 日期：YYYY-MM-DD
  来源文件：文件路径或 user-chat:YYYY-MM-DD
  用户原话摘要：用户内容精简版
  分类：中文分类
  内容：caveman 风格记忆
  状态：生效 / 待确认 / 已废弃 / 已替换
  替代：无或被替代记忆摘要
```

### 写入约束

- **强制前置步骤**：写入 `memory.md` 前，必须执行 `node -e "console.log(new Date().toISOString().split('T')[0])"` 获取当前系统日期，再将输出值填入 `日期` 字段。禁止跳过此步骤，禁止从上下文、已有记忆条目、对话历史中复制日期。

### 冲突处理

当用户当前内容与旧记忆冲突时，Agent 不自动覆盖旧记忆。

必须列出：

- 旧记忆。
- 用户新说法。
- 冲突点。

然后向用户确认哪个正确。用户确认前，不更新旧记忆，不写入冲突内容，不写进化日志。

用户确认后，再更新 `memory.md`，并写入 `evolution-log.md`。

## 知识库

知识库目录：`docs/knowledge/`

知识库必须申请，确认后才能落盘。

### 触发条件

满足以下至少一类时，Agent 应发起知识库进化申请：

- 内容长期有效。
- 内容可复用。
- 内容是项目事实。
- 内容是项目背景信息。
- 内容会影响后续开发判断。

### 申请格式

```md
[进化申请-知识库]
原因：
来源：
拟写入：
目标文件：
内容摘要：
风险：
需要用户确认：是
```

未获确认前，只能在回复中提供申请和草案，不得创建或修改知识库文件。

## 技能

skill 创建或更新必须申请，确认后才能落盘。

### 触发条件

满足以下至少一类时，Agent 应发起技能进化申请：

- 同类任务重复出现 2 次以上。
- 现有流程每次都需要大量人工纠偏。
- 用户明确说“以后都按这个做”。
- 某个工作流需要固定产物格式和步骤。
- 现有 skill 与本项目 SDD 规范冲突。

### 不触发

- 一次性任务。
- 普通代码风格偏好。
- 还不稳定的探索过程。
- 写 README 或知识库即可解决的问题。

### 申请格式

```md
[进化申请-技能]
触发原因：
证据：
拟修改对象：
拟修改内容：
影响范围：
风险：
需要用户确认：是
```

未获确认前，只能在回复中提供申请和草案，不得创建或修改 skill 文件。

## 进化日志

进化日志文件：`docs/agent/evolution-log.md`

只记录已执行进化。被驳回或未确认的申请不写日志。

记忆自主落盘后，必须同步写简短日志。

格式：

```md
- YYYY-MM-DD｜类型｜动作｜目标文件｜摘要
```

### 写入约束

- **强制前置步骤**：写入 `evolution-log.md` 前，必须执行 `node -e "console.log(new Date().toISOString().split('T')[0])"` 获取当前系统日期，再将输出值作为日志日期。禁止跳过此步骤，禁止从上下文、已有日志条目、对话历史中复制日期。
