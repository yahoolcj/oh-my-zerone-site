## 使用场景

当你使用的编码模型（如 DeepSeek、GLM）不具备识图能力时，用 `/image-to-context` 将截图/设计稿/报错/架构图通过外部视觉模型 API 转化为结构化文本上下文，自动注入当前对话，后续可直接 `/to-coding` 开发。

> **前置依赖**：需要 `zerone/cli >= 0.8.0` 并执行 `zerone omz set vision-apikey` 配置视觉模型 API Key。详见下方配置指南。

## 使用方式

```text
/image-to-context <场景描述>
```

场景描述用自然语言即可，skill 会自动匹配输出模板。

## 支持的图片类型

| 场景 | 示例描述 | 输出内容 |
|------|---------|---------|
| UI 设计稿 | "这是登录页截图" | 布局、组件清单（含属性）、颜色间距、交互状态、标注说明 |
| 代码报错 | "控制台报错截图" | 错误原文、文件路径、堆栈关键帧、推断原因及置信度 |
| 架构图 | "这是系统架构图" | 节点清单、关系连线、数据流方向、约束 |
| 通用描述 | "帮我看看这张图" | 图片类型判断、完整内容描述、编码相关信息 |

## 提示

- 支持多图同时上传，结果按 `## 图1` `## 图2`... 逐张输出。
- 图片中的框选、箭头、圈画等标注会被显式识别并标记 `⚠️`。
- 不落盘 —— 识别结果仅注入对话上下文，不会在项目目录创建文件。
- 没有图片数量硬限制，实际可用数量取决于视觉模型的上下文窗口。
