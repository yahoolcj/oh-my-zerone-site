## 使用场景

开发完成后、提交代码前，用 `/to-test` 对照 PRD 的 TODOQ 逐条验收回收。Agent 会逐项确认每一条功能是否通过/未通过/待确认，输出精简的测试回收报告。如果某条未通过，继续定位修复。

## 使用效果示例

```terminal
$ /to-test

> 对照 .PRD/2.1.0.md TODOQ 逐项验收:

> [✓] #1 新增 SHOWCASE.md 数据源 — 通过: 34 个文件全部存在
> [✓] #2 扩展 skills 数据层 — 通过: getAllSkills() 返回 scenario/snippet
> [✓] #3 列表页改版 — 通过: 2列 grid + category 筛选正常
> [!] #4 详情页改版 — 待确认: 终端样式在 light mode 下对比度偏低
> [ ] #5 i18n 补充 — 待测试
> [ ] #6 清理旧代码 — 待测试

> 测试回收报告:
> 3/6 通过, 1/6 待确认, 2/6 待测试
> 建议: 修复 #4 light mode 对比度后重新验收。
```
