## 使用场景

当你已经有一份初步的需求文档、设计草稿或任意文本内容，但感觉细节不够扎实、存在模糊地带时，用 `/grill-from-draft` 让 Agent 从这份已有内容出发进行深度追问。

与另外两个 grill 技能的区别：
- `/grill-me`：从零开始，适合你只有一个想法、还没有任何书面材料时
- `/grill-from-draft`（本技能）：从已有文档出发，先补全文档内的模糊点，再发现文档未覆盖的新分支
- `/grill-with-docs`：基于项目已有的 CONTEXT.md 和 ADR 进行方案压测，适合需要与项目领域模型对齐时

输入方式支持两种：
1. **文件路径**：`/grill-from-draft ./需求草稿.md` → 最终产出 `需求草稿_grilled.md`
2. **粘贴内容**：`/grill-from-draft` 然后粘贴文本 → 会话内汇总共识

## 使用效果示例

### 文件路径输入

```terminal
$ /grill-from-draft ./需求草稿.md

> 已读取 需求草稿.md，识别到以下问题：

> Phase 1 - Question 1/?: 文档提到"用户管理"但未定义具体范围。推荐理解：用户注册、登录、权限分配。是否确认？
You: 还要加上用户信息编辑和注销

> Phase 1 - Question 2/?: 第3行说"支持导出Excel"，第12行说"支持导出CSV"——这两个是一个功能还是分开的？
You: 都要支持，Excel和CSV两种格式

> Phase 1 文档完整性补全完毕。进入 Phase 2

> Phase 2 - Question 3/?: 文档没有提到权限控制，是否需要区分管理员和普通用户的操作权限？
You: 需要，管理员可以管理所有用户，普通用户只能编辑自己的信息

> Phase 2 - Question 4/?: 导出功能是否需要异步任务（数据量大时）？
You: 不需要，数据量不大

> 设计树已全部走完。已生成 需求草稿_grilled.md，内容概要：
> - 顶部引用原文链接
> - 用户管理：注册、登录、权限分配、信息编辑、注销 ✓
> - 导出格式：Excel + CSV ✓
> - 权限控制：管理员（全部用户）、普通用户（仅自己） ✓
> - 导出方式：同步导出 ✓
> 确认后进入下一步工作流。
```

### 粘贴内容输入

```terminal
$ /grill-from-draft

You: [粘贴了一段：我想做一个消息通知中心，用户能收到系统通知和业务通知]

> 已接收粘贴内容，识别到以下问题：

> Phase 1 - Question 1/?: "系统通知"和"业务通知"的区别是什么？推荐理解：系统通知=平台级（维护公告等），业务通知=业务操作触发的（订单状态变更等）
You: 对

> Phase 1 - Question 2/?: 通知的送达方式是站内消息还是需要推送（App推送/邮件/短信）？
You: 先只做站内消息

> 设计树已走完。共识汇总如下：
> - 通知类型：系统通知（平台级）+ 业务通知（订单/工单等业务操作触发）
> - 送达方式：仅站内消息
> 确认后进入下一步工作流。
```
