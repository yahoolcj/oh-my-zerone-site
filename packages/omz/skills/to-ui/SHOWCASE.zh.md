## 使用场景

PRD 完成后，想在编码前先看到页面的视觉效果和交互原型，用 `/to-ui` 基于 PRD 生成独立的 HTML 原型文件并在浏览器中预览。它使用项目 z1-design 设计规范配色、Tailwind CSS 布局、GSAP 动画增强，产出可直接打开预览的 HTML。

## 使用效果示例

```terminal
$ /to-ui .PRD/2.1.0.md

> 解析 PRD 数据结构与交互规则...
> 生成 UI 原型: .UI/2.1.0-skills-showcase.html
>
> 布局预览:
> ┌─────────────────────────────────────┐
> │ [全部] [clarify] [plan] [develop] ... │  ← category 筛选栏
> ├──────────────────┬──────────────────┤
> │ ╭─ grill-me ────│ ╭─ to-prd ──────│
> │ │ 🔵 clarify     │ │ 🟣 plan        │
> │ │ 持续追问方案... │ │ 整理上下文...   │
> │ │ ┌─terminal──┐ │ │ ┌─terminal──┐ │
> │ │ │ $ /grill.. │ │ │ │ $ /to-prd │ │
> │ │ └───────────┘ │ │ └───────────┘ │
> │ ╰───────────────│ ╰───────────────│
> └──────────────────┴──────────────────┘
>
> 已自动打开预览。确认后进入 /to-coding。
```
