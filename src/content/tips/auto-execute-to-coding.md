`/to-coding` 默认行为是每完成一个 TODO 项就停下来等待用户验收。这在需求不明确时需要频繁对齐的场景下很实用，但如果你已经对需求有清晰认识，每次停顿反而打断节奏。

## 授权方式

在执行 `/to-coding` 时，直接告诉 Agent：

```
继续编码，结束后停止
```

或者更精确地指定范围：

```
自动执行到 TODO 第 5 条
```

Agent 收到授权后会连续执行，暂时跳过中间的确认环节，直到达到指定目标后才停下来汇报所有细节。

## 授权有效期

- **无目标指定**（"继续编码，结束后停止"）：授权持续到当前 step 完成，然后自动消除
- **指定目标**（"自动执行到 TODO 第 5 条"）：执行到对应目标后授权自动消除，回到确认模式

## 随时中止

任何时候你都可以打断自动执行：

```
停下来
暂停
不要自动执行
我要确认下是否正确
```

Agent 收到后会立即移除授权，回到逐项确认模式。

## 适用场景

| 场景 | 建议 |
|------|------|
| 需求清晰，知道最终效果 | 使用自动执行，节省等待时间 |
| 新功能探索，需要频繁调整方向 | 保持默认逐项确认 |
| 多个简单、独立的 TODO 项 | 一次性授权全部执行 |
| 涉及复杂业务逻辑 | 至少在前 1-2 步确认后再授权后续 |

## 注意事项

自动执行不会降低代码质量——Agent 仍然遵循 `rules/coding.mdc` 中的所有约束，只是跳过了中间的确认步骤。执行完成后依然需要统一的验收和测试回收。
