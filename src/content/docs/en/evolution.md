## The lobster in your project

Not a one-off AI contractor. Be the lobster in your project: claws for Skills constraints, shell for permission boundaries, growth through approved evolution.

The lobster stays in the repo, learns team language, and gets steadier over time. omz does not let Agents rewrite everything freely.

## What evolution is not

Evolution is not unchecked self-modification. Agents may spot opportunities, gather evidence, and file proposals. Whether anything is written, where it goes, and which long-term assets are affected must follow `docs/agent/evolution.md`.

## Typical evolution targets

- `docs/agent/memory.md`: long-term memory and project conventions
- `docs/knowledge/`: reusable project knowledge
- `skills/`: reusable task capabilities
- `rules/`: long-term execution rules
- `docs/agent/evolution-log.md`: completed evolution actions

## Memory vs Knowledge

Memory stores short, precise, long-lived execution guidance. Knowledge stores project facts and domain context that get referenced repeatedly. Knowledge updates are heavier and require user confirmation.
