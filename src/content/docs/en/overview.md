## What is oh-my-zerone?

`oh-my-zerone` (omz) is a **Harness Engineering** suite built on SDD (Specification Driven Development) for frontend AI Coding workflows.

It does not lock you to one AI editor. Instead, it splits clarification, planning, PRD, issues, coding, testing, quality checks, review, and commit/PR into composable rules and skills.

## What does it solve?

It helps frontend teams deliver with AI more reliably.

Without workflow guardrails, Agents jump from a vague prompt to code, causing scope creep, inconsistent terms, and unclear acceptance criteria. omz standardizes these collaboration stages so Agents follow project language, docs, and workflow.

Good fits:

- Scattered requirements that need structured specs
- Changes touching API, interaction, docs, types, and tests
- Teams that need Agents to avoid skipping steps or unauthorized refactors
- Clear TODOs before coding and item-by-item acceptance after

## Glossary

- **SDD**: Specification Driven Development. Define boundaries and acceptance before coding.
- **ADR**: Architecture Decision Record for tracing major technical choices.
- **AFK**: Away From Keyboard. Tasks Agents can push without constant human input.
- **HITL**: Human In The Loop. Steps that require human confirmation or acceptance.
