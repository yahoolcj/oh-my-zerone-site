## 做项目里的龙虾

不做一次性外包 AI，做项目里的龙虾：有钳（Skills 约束）、有壳（权限边界）、会进化（授权沉淀）。

龙虾长期待在项目里，熟悉团队语言与协作习惯。omz 不是让 Agent 随意自改，而是让它在授权边界内越用越稳。

## 进化不是什么

进化不是 Agent 想改什么就改什么。Agent 只能识别机会、整理证据、发起申请。是否落盘、写入哪里、影响哪些长期资产，都必须遵循 `docs/agent/evolution.md` 的授权边界。

## 典型进化对象

- `docs/agent/memory.md`：沉淀长期记忆与项目约定
- `docs/knowledge/`：沉淀可复用项目知识
- `skills/`：沉淀可复用任务能力
- `rules/`：沉淀长期执行规则
- `docs/agent/evolution-log.md`：记录已完成的进化动作

## Memory 与 Knowledge 的区别

Memory 记录短、准、长期有效的执行依据。Knowledge 记录需要被多次引用的项目事实与领域背景。知识库属于更重的长期资产，更新前需要用户确认。
