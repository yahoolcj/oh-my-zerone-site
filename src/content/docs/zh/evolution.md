## 自主进化机制

越用越稳定：沉淀记忆、知识与能力，但不绕过授权边界。

Agent 是项目长期负责人，可主动识别进化机会、整理证据、发起申请；只有在授权或用户确认后，才能落盘。

不做一次性外包 AI，做项目里的龙虾：有钳（Skills 约束）、有壳（权限边界）、会进化（授权沉淀）。

## 为什么要进化

AI 不能每次都从零开始。Agent 是项目长期负责人，值得把约定、背景和协作习惯留下来，下次任务直接复用。

长期待在项目里，Agent 会熟悉团队语言与协作习惯。omz 不是让 Agent 随意自改，而是让它在授权边界内越用越稳。

## 怎么进化

进化不是 Agent 想改就改。识别机会、整理证据、等你点头，才写入长期资产。

1. **识别进化机会**：对话中发现值得长期保留的约定、事实或能力缺口。
2. **整理进化申请**：整理证据、来源与拟写入位置，形成可确认的提案。
3. **你确认授权**：知识库、Skill、规则等项目级资产须你确认；memory 可按授权边界自主落盘。
4. **写入长期资产**：确认后写入 memory、knowledge、skill、rule 或 evolution-log。

主动识别 memory、knowledge、skill、rule、文档等值得沉淀的内容，严格遵循 `docs/agent/evolution.md` 授权边界。

## 进化需要注意什么

未验证的猜测不能当事实。知识库、Skill、规则改动须你确认。密钥和隐私绝不落盘。

### 进化授权边界

**可自主执行**

- 每次对话时识别进化机会
- 整理进化申请
- 自主写入、更新、废弃 memory
- 记忆落盘后记录简短进化日志

**需要用户确认**

- 添加或更新 `docs/knowledge/` 知识库
- 创建或更新 skill
- 修改长期规则、工作流、`AGENTS.md` 等项目级约束

**禁止执行**

- 未确认前创建或修改知识库、skill 文件
- 把未经验证的猜测写成项目事实
- 把一次性任务细节写成长期规则
- 删除或覆盖历史规则来扩大自身权限
- 记录密钥、账号、token、隐私数据等敏感信息

## 进化触发节点

1. 跟用户深度讨论后，获得长期有效的新信息
2. 用户在工作过程中提出重点背景、协作原则、长期偏好
3. 用户反驳或纠正 Agent 已形成的记忆
4. 内容中明确出现 `[进化-知识库]`、`[进化-技能]`、`[进化-记忆]` 等标记
5. 现有能力无法稳定完成某类重复任务
6. 用户多次纠正同一 skill 的执行产物

## 进化落盘对象

### 记忆层沉淀

文件：`docs/agent/memory.md`

保存经授权的长期偏好、项目约定与协作规则，让后续任务更一致。

### 知识库扩展

目录：`docs/knowledge/`

沉淀长期有效、可复用的项目事实与背景，减少重复解释和临时猜测。

### Skills & Rules

目录：`skills/`、`rules/`

识别值得沉淀的 skill、rule 与文档，遵循 `docs/agent/evolution.md` 授权边界后落盘。

## Memory 与 Knowledge 的区别

Memory 记录短、准、长期有效的执行依据。Knowledge 记录需要被多次引用的项目事实与领域背景。知识库属于更重的长期资产，更新前需要用户确认。

## 记忆冲突处理

当用户当前内容与旧记忆冲突时，Agent 不自动覆盖旧记忆。

必须列出：

- 旧记忆
- 用户新说法
- 冲突点

然后向用户确认哪个正确。用户确认前，不更新旧记忆，不写入冲突内容，不写进化日志。

用户确认后，再更新 `memory.md`，并写入 `evolution-log.md`。

## 延伸阅读

完整授权机制、申请格式与日志规范见仓库内 `docs/agent/evolution.md`。
