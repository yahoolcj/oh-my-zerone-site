# 2.1.0

## Skills 篇章展示改版

> 类型：优化升级

### 概述

网站 Skills 篇章当前直接展示 SKILL.md 原内容（给 AI 看的 prompt 指令），对网站访客不友好。本次改为面向人类用户展示每个 skill 的 **使用场景** 和 **使用效果示例**，让访客快速理解"什么时候用它、用了之后交互长什么样"。

内容来源从 SKILL.md 切换为新的 SHOWCASE.zh.md / SHOWCASE.en.md，列表页和详情页同步改版。

| slug | - | string | 是 | - | skill 目录名 |

| showcaseEn | - | string | 是 | - | 英文完整 SHOWCASE markdown |

#### SHOWCASE.md 文件结构约定
- 语言独立：`SHOWCASE.zh.md` / `SHOWCASE.en.md`
- 内容结构：
  - `## 使用场景` / `## Use Scenarios` — 自由段落，第一段自动作为列表页摘要
  - `## 使用效果示例` / `## Example Effects` — 包含至少一个伪终端代码块，第一个代码块自动作为列表页效果片段

### 页面交互

- 列表页顶部 category 筛选标签栏（clarify / plan / develop / review / utility 五色标签），点击切换筛选，默认显示全部
- 列表页卡片 hover 有微动效（边框高亮 + 微缩放），卡片内代码片段以暗色终端风格渲染（`bg-[var(--color-surface)]` + mono 字体）
- 详情页"使用效果示例"区块以暗色终端外观包裹（类似 HeroTerminal 风格）
- category 标签颜色映射：clarify=蓝, plan=紫, develop=绿, review=橙, utility=灰
- 动效：卡片入场 stagger（Motion whileInView），尊重 `prefers-reduced-motion`
- 中英双站分开展示对应语言内容

### 扩展性要求

- SHOWCASE.md 放在各 skill 目录下（与 SKILL.md 同级），skill 作者可自行维护
- 解析逻辑靠 markdown 结构约定，不依赖 frontmatter 等额外配置
- lib/skills.ts 的 `getAllSkills()` 返回数据结构向后兼容，仅扩展字段

### 用户故事

1. 作为网站访客，我打开 Skills 列表页，可以通过 category 筛选快速定位到"开发阶段"的 skill，并通过卡片中的场景摘要判断这个 skill 是否是我需要的
2. 作为网站访客，我点击某个 skill 进入详情页，可以先读"使用场景"理解何时触发，再通过"使用效果示例"中的终端对话模拟直观感受交互过程
3. 作为 skill 作者，我只需要在 skill 目录下新增 SHOWCASE.zh.md 和 SHOWCASE.en.md，无需修改其他配置，网站自动展示
