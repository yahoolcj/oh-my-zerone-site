# 2.2.0

## grill-from-draft Skill

> 类型：新增

### 概述

创建 `grill-from-draft` skill，作为 `grill-me` 的场景补充。`grill-me` 适用于从 0 到 1 的设计澄清，而 `grill-from-draft` 适用于用户已有一份初步需求文档/设计文档/任意内容，需要从该内容出发进行深度追问的场景。

核心能力：先对文档内容进行理解和 grill，补全文档完整性（模糊术语、缺失决策、内部矛盾），再以 grill-me 的思维发现未覆盖的新分支并逐条澄清。

输入支持文件路径和粘贴内容两种形式。文件路径输入时产出 `*_grilled.md`（顶部引用原文链接，正文为 reorganized + grilled 结论）；粘贴内容输入时在会话内汇总。

### 页面交互

不涉及前端页面，纯 skill 文件创建与文档更新。

### 扩展性要求

- `grill-from-draft` 与 `grill-me`、`grill-with-docs` 保持同级，共享 grill 通用追问模式（每次一问 + 推荐答案）
- 不依赖项目必须存在 CONTEXT.md 或 ADR
- `*_grilled.md` 作为中间产物，可直接作为 `to-prd` 的输入
- 文件命名规则：`{原文件名}_grilled.md`，统一后缀

### 用户故事

1. 用户已有一份 `需求草稿.md`，执行 `/grill-from-draft ./需求草稿.md`，Agent 读取文档后逐条追问模糊术语、补全缺失决策、指出内部矛盾，最终生成 `需求草稿_grilled.md`
2. 用户粘贴一段设计思路文本，执行 `/grill-from-draft`，Agent 基于粘贴内容进行两阶段追问，会话内汇总共识结论
3. 用户草稿中有两处矛盾说法，Agent 在 Phase 1 识别并指出，由用户澄清后统一写入 grilled 文件
4. 用户草稿未涉及部署方案，Agent 在 Phase 2 主动提出新分支追问，用户确认后写入 grilled 文件
5. grilled 文件顶部包含指向原始文档的引用链接，正文按 reorganized 结构组织
