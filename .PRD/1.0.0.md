# 1.0.0

## oh-my-zerone 门户网站

> 类型：新增

### 概述

为 `oh-my-zerone`（简称 `omz`）AI Coding 套件建设一个公网宣传门户网站，面向公司内部前端开发者，帮助其快速了解套件价值、完成安装上手、查阅 Skills 文档。站点采用 Astro 静态站点方案，视觉气质参考 [Taste Skill](https://www.tasteskill.dev/)，由 `design-taste-frontend` 技能自主定品牌视觉。v1 仅做宣传，不包含工作流介绍、Testimonials、案例展示等模块。

### 数据模型 Schema

#### SiteConfig

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| version | 文本 | string | 是 | `1.2.0` | omz 套件版本号，展示于 Footer |
| siteName | 文本 | string | 是 | `oh-my-zerone` | 站点名称 |
| siteAbbr | 文本 | string | 是 | `omz` | 简称，用于 Logo 文字标 |
| defaultLocale | 选择 | `'zh' \| 'en'` | 是 | `zh` | 默认语言 |
| heroInstallCmd | 文本 | string | 是 | `zerone setup omz` | 首页 Hero 展示的安装命令（不含 registry） |
| docsInstallCmdFull | 代码块 | string | 是 | - | `/docs/install` 中文页完整安装命令（含内网 registry） |

#### EditorItem

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| id | 文本 | string | 是 | - | 唯一标识，如 `cursor` |
| name | 文本 | string | 是 | - | 显示名称，如 `Cursor` |
| iconSlug | 文本 | string | 是 | - | Simple Icons / devicon slug |
| url | 链接 | string | 否 | - | 编辑器官网链接 |

#### SkillCard

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| slug | 文本 | string | 是 | - | 目录名，如 `to-coding` |
| name | 文本 | string | 是 | - | skill 名称，保持英文 |
| descriptionZh | 文本 | string | 是 | - | 中文一句话描述 |
| descriptionEn | 文本 | string | 是 | - | 英文一句话描述 |
| category | 选择 | string | 否 | - | 可选分类：clarify / plan / develop / review / utility |
| sourcePath | 文本 | string | 是 | - | `.omz/skills/[slug]/SKILL.md` 路径 |

### 页面交互

- 首页 Hero 安装命令支持点击复制，复制成功后按钮文字短暂变为「已复制」/ `Copied`
- 顶部导航 `中 / EN` 切换时，保持当前页面路径对等跳转（如 `/docs/skills` ↔ `/en/docs/skills`）
- Skills 卡片 hover 时有轻微边框/背景变化，点击跳转至对应 `/docs/skills/[slug]` 详情页
- Docs 侧栏当前页高亮，移动端侧栏折叠为汉堡菜单
- 首页各区块进入视口时触发 scroll-reveal 动画（`MOTION_INTENSITY: 6`），`prefers-reduced-motion` 时降级为静态
- 首页不做工作流介绍，不出现 Testimonials、案例墙、Sponsor、Changelog 独立区块
- 公网首页不展示内网 registry 地址（`10.0.0.74:8081`），完整安装步骤仅在 `/docs/install` 中文页展示
- 英文 `/en/docs/install` 使用通用 npm 安装描述，registry 引导为 "use your team's internal registry"
- skill 名、CLI 命令、文件路径、代码块在两种语言下均保持英文原文

### 扩展性要求

- Content Collections 与 `.omz/skills/` 扫描机制需支持后续 skill 增删自动同步，无需改页面代码
- i18n 路由架构预留扩展第三语言的可能，但 v1 仅中英文
- 工作流介绍（`/docs/workflow`）、Changelog、Testimonials、案例墙等模块明确排除在 v1 外，架构上预留路由扩展空间
- 品牌视觉使用 CSS 变量/token，便于后续对齐 zerone 公司 VI
- `/docs/skills/[slug]` 英文详情 v1 可后补，路由与数据结构需提前兼容
- 域名与 CI 自动部署配置待定，部署方案需支持 Astro 静态输出

### 用户故事

1. 作为公司内部前端开发者，我希望打开门户网站后 30 秒内了解 omz 是什么、解决什么问题，以便判断值不值得安装。
2. 作为尚未安装 omz 的同事，我希望在首页看到一键复制的 `zerone setup omz` 命令，以便快速开始安装。
3. 作为已安装 omz 的开发者，我希望在 Skills 目录中查阅某个 skill（如 `/to-coding`）的详细说明，以便正确使用它。
4. 作为需要完整安装步骤的同事，我希望在 `/docs/install` 中文页看到含内网 registry 的完整命令，以便顺利完成环境配置。
5. 作为英文阅读习惯的开发者，我希望通过顶部语言切换查看英文版首页和文档，以便理解套件能力。
6. 作为团队推广者，我希望站点公网可访问且视觉专业（参考 Taste Skill），以便向更多前端同事分享链接。
