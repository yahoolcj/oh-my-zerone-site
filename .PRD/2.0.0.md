# 2.0.0

## Monorepo 合并：oh-my-zerone 并入 omz-website

> 类型：优化升级

### 概述

将 oh-my-zerone（skills/rules/docs 配置仓库）合并到 omz-website（Astro 网站项目）中，统一为 monorepo 进行维护。oh-my-zerone 内容放入 `packages/omz/` 目录。同时修改 @zerone/cli 的 `zerone setup omz` 命令，指向新的 monorepo 仓库地址，并通过 sparse checkout 只拉取 `packages/omz/` 目录进行分发。

### 扩展性要求

- 根目录 `AGENTS.md` / `docs/agent/` 与 `packages/omz/` 下同名文件**各自独立维护**：
  - 根目录 → 约束网站本身开发
  - `packages/omz/` → 通过 CLI 分发给用户项目
- `packages/omz/` 不设独立 package.json，只是一个文件目录
- 未来若 omz-kit 需要独立 npm 发布，再补加 package.json

### 用户故事

1. 作为 omz 维护者，我只需在一个仓库里同时维护网站和 omz-kit，不用在两个仓库之间切来切去。
2. 作为用户，执行 `zerone setup omz` 时从新仓库地址拉取，安装行为与之前完全一致。
3. 作为用户，访问网站看到的安装命令会自动指向新仓库，文档内容保持一致。
