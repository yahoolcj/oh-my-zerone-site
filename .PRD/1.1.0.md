# 1.1.0

## 使用技巧栏目

> 类型：新增

### 概述

在 oh-my-zerone 门户网站新增「使用技巧」栏目，帮助开发者快速掌握 omz 套件的实用技巧。首页以横向列表形式展示技巧预览，Docs 下新增 `/docs/tips` 索引页（卡片网格）和 `/docs/tips/[slug]` 详情页（Markdown 正文渲染）。支持中英文双语路由，数据层采用 Astro Content Collections，后续可持续添加新技巧无需修改页面代码。

### 数据模型 Schema

#### UsageTip（Content Collection frontmatter）

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| titleZh | 文本 | string | 是 | - | 技巧中文标题 |
| titleEn | 文本 | string | 是 | - | 技巧英文标题 |
| summaryZh | 文本 | string | 是 | - | 中文一句话摘要（≤30字） |
| summaryEn | 文本 | string | 是 | - | 英文一句话摘要 |
| order | 数字 | number | 是 | 99 | 排序权重，升序排列 |

#### UsageTipListItem（首页/索引页展示用）

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| slug | 文本 | string | 是 | - | 文件名即 slug，如 `quick-start-strangers` |
| order | 数字 | number | 是 | - | 来自 frontmatter |
| titleZh | 文本 | string | 是 | - | 来自 frontmatter |
| titleEn | 文本 | string | 是 | - | 来自 frontmatter |
| summaryZh | 文本 | string | 是 | - | 来自 frontmatter |
| summaryEn | 文本 | string | 是 | - | 来自 frontmatter |
| href | 链接 | string | 是 | - | 本地化路径，如 `/docs/tips/xxx` |
| index | 数字 | number | 是 | - | 列表序号 1-based |

#### ProseArticle

| 字段 | 控件 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|------|--------|------|
| html | 插槽 | string | 是 | - | renderMarkdown 输出 |
| header | 具名插槽 | slot | 否 | - | 标题区域自定义 |

### 页面交互

- 首页 UsageTips 区块展示前 N 条技巧（默认 6），横向列表形式，每行包含序号、标题、摘要（单行截断）和 → 箭头
- 点击技巧行/箭头跳转至对应 `/docs/tips/[slug]` 详情页
- 首页区块标题右侧「查看全部技巧 →」链接至 `/docs/tips`
- `/docs/tips` 索引页以卡片网格展示全部技巧，每条卡片含标题 + 摘要，hover 高亮，点击进详情
- `/docs/tips/[slug]` 详情页渲染 Markdown 正文，顶部有「← 返回技巧列表」链接
- Docs 侧栏当前页高亮，`/docs/tips` 及子路由下 `activeSlug` 为 `'tips'`
- 技巧卡片和首页列表进入视口时触发入场动画（具体效果由 design-taste-frontend 决定），`prefers-reduced-motion` 时降级为静态
- 语言切换保持路径对等跳转（`/docs/tips/xxx` ↔ `/en/docs/tips/xxx`）

### 扩展性要求

- Content Collection 扫描机制支持后续新增技巧 `.md` 文件自动同步，无需改页面代码
- `limit` prop 支持首页展示数量灵活调整
- 中英文路由架构已预留，后续可补英文详情正文（frontmatter 已含 titleEn/summaryEn）
- ProseArticle 组件设计为通用 prose 容器，后续其他文档页可复用

### 用户故事

1. 作为已安装 omz 的开发者，我希望在首页看到实用的使用技巧，以便快速提升使用效率。
2. 作为想深入了解某个技巧的开发者，我希望点击技巧后进入详情页查看完整说明，以便完整掌握操作方法。
3. 作为英文用户，我希望通过语言切换查看英文版技巧列表和详情，以便无障碍阅读。
4. 作为内容贡献者，我只需在 `src/content/tips/` 下新增 Markdown 文件即可添加新技巧，无需修改任何页面代码。
5. 作为文档浏览者，我希望能从 Docs 侧栏直接进入使用技巧，与其他文档入口保持一致的导航体验。
